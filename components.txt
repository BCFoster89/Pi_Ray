THRUSTER MOTOR DRIVERS
────────────────────────────────────────
Board         : ANMBEST Dual High-Power MOSFET Trigger Switch Drive Module
Part #        : (ASIN B07NWD8W26 or similar — confirm on package)
Input voltage : DC 5 V–36 V
Cont. current : 15 A per channel
Peak current  : 30 A per channel
Power rating  : 400 W
PWM input     : 0–20 kHz signal (Pi GPIO 3.3 V logic compatible)
Type          : Single-direction MOSFET switch (NOT an H-bridge — each board drives one direction only)
Channels      : Dual (two independent MOSFET switches per board)

GPIO assignments (BCM numbering):
  Surge fwd   : GPIO  (confirm in config.py)
  Surge rev   : GPIO  (confirm in config.py)
  Sway fwd    : GPIO  (confirm in config.py)
  Sway rev    : GPIO  (confirm in config.py)
  Ascend      : GPIO 15, 18
  Descend     : GPIO 6, 20
  Yaw         : GPIO  (confirm in config.py)

Software PWM  : gpiozero PWMOutputDevice, 200 Hz
Ramp rate     : 0.05 duty/update (0→100% in ~1 s at 20 Hz command rate)

KNOWN ISSUE — Vertical thruster (ascend/descend):
  One motor is driven by two single-direction boards (one wired reversed).
  The inactive board's protection diodes block the reverse current path.
  Hardware fix needed: replace with proper H-bridge (L298N, DRV8833, TB6612)
  or add 1N4001 isolation diodes in series with each board's motor output.

BROWNOUT RISK:
  Motor inrush current (locked-rotor) can sag battery voltage below Pi minimum (4.75 V).
  Mitigations (in order of importance):
    1. [HARDWARE] Power Pi from separate 5 V supply (BEC or power bank) — isolate from motor battery
    2. [HARDWARE] Add 1000 µF / 35 V+ electrolytic cap across each board's V+ and GND
    3. [HARDWARE] Add 0.1 µF ceramic cap in parallel with electrolytic (high-freq suppression)
    4. [HARDWARE] Use ≥16 AWG (14 AWG preferred) wire for motor supply runs
    5. [HARDWARE] Add 1N5819 Schottky flyback diode across each motor terminal if not built in
    6. [SOFTWARE]  ramp_rate = 0.05 slows motor startup to ~1 s (already applied)
