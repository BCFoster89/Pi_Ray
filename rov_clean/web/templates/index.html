Sensor Suite (9-DOF)

        
Three sensors share the I2C bus. The system degrades gracefully — if a sensor is missing, the filter falls back automatically.

        

          
LSM6DSO (0x6B)Accelerometer + gyroscope — roll, pitch, angular rate

          
LPS28 (0x5D)Barometric pressure — depth in feet

          
MMC5603 (0x30)Magnetometer — absolute compass heading (yaw reference)

        


        
Attitude Filter (Madgwick)

        
Replaces the old complementary filter. Uses gyro + accel + mag together as a quaternion to produce drift-free roll, pitch, and yaw at 20 Hz.

        

          
Beta (β)Controls how aggressively the filter corrects toward accel/mag. Lower = smoother but slower to level. Default 0.1; try 0.033 if the horizon jumps.

          
Tune via APIPOST /imu_tune {"beta": 0.033} — takes effect immediately, no restart.

          
6-DOF fallbackIf MMC5603 is absent, filter runs without mag — yaw will drift over time.

        


        
Dead Reckoning (DR) Map

        
Runs server-side at 20 Hz using quaternion rotation to remove gravity before integrating acceleration into velocity and position. Accuracy degrades over minutes without an external position fix.

        

          
Damp (0.80–1.00)Velocity decay per update. Higher = more momentum/glide; lower = stops quickly.

          
Dead (0.00–1.00)Minimum acceleration threshold (m/s²). Higher = ignores small vibrations and noise.

          
Scale (0.1–5.0)Sensitivity multiplier. Higher = map moves more per unit of movement.

          
ResetClears position to origin and erases trail on both server and map.

        


        
Depth Hold PID

        
PID controller drives the vertical thrusters to maintain a target depth. Use Go To to set a specific depth in feet.

        

          
Kp (0–5)Proportional gain — corrects depth error. Too high causes oscillation.

          
Ki (0–2)Integral gain — corrects slow persistent drift. Too high causes slow oscillation.

          
Kd (0–2)Derivative gain — dampens overshoot. Too high amplifies sensor noise.

          
Go ToEnables depth hold and sets target to the entered value (ft).

        


        
Heading Hold PID

        
Locks the ROV to its compass heading when enabled. Manual yaw input is ignored while active. Handles 360° wrap-around automatically.

        

          
Kp (0–10)Proportional gain — how hard it fights heading error. Default 1.5.

          
Ki (0–2)Integral gain — corrects slow yaw bias. Default 0.05.

          
Kd (0–5)Derivative gain — dampens yaw oscillation. Default 0.3.

          
Requires magWorks best with MMC5603 connected and calibrated. In 6-DOF mode yaw drifts, so heading hold will slowly drift too.

        


        
Calibration

        

          
Zero DepthSets the current pressure reading as the 0 ft reference.

          
Zero HorizonResets the Madgwick quaternion to identity — re-levels roll and pitch from the current position.

          
Zero IMUResets gyro/accel bias offset (hold ROV still first).

          
Toggle LEDTurns the ROV's LED lighting on or off.

          
Mag CalCollects ~500 magnetometer samples while you rotate the ROV through all orientations, then fits hard-iron offsets and soft-iron scale. Saves to calibration.json. Do this before using heading hold.

        


        
Camera Tilt

        
Servo on GPIO 18 (hardware PWM, 50 Hz). Push left stick up to tilt camera up, down to tilt camera down. Servo returns to center on E-stop.

        

          
Center7.5% duty cycle — neutral position (~1.5 ms pulse).

          
Range±1.5% duty from center — approx ±27° of tilt.

          
TuneAdjust SERVO_CENTER and SERVO_RANGE in servo.py if the neutral position or travel range needs correction.

        


        
Camera Focus

        

          
AutoContinuous autofocus — camera adjusts focus automatically.

          
ManualManual lens position via slider. 0 = infinity (far), 10 = closest macro.

          
AF OnceSingle autofocus shot, then locks focus in place.
